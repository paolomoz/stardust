What it is

Not a generator. A pipeline you drive.

Stardust is a set of skills you invoke in order. Each one turns a decision into an artifact — a brand profile, a briefing, a wireframe, a design, a block — and writes it to disk where you can read it, edit it, re-run it.

No brand yet? Stardust helps invent one. Already have one? It extracts from the live site. Skip a step, re-run a step, branch off where it makes sense. The pipeline bends around the project, not the other way round.

Nothing happens in a black box. Every output is a file you can open.

The pipeline

Seven skills, two phases.

The design phase is platform-agnostic — it'd port to a different SSG tomorrow. The EDS phase turns approved designs into a real aem-boilerplate project: lintable, deployable, no lock-in.

Design phase · 01 – 04 EDS phase · 05 – 07
01 · /stardust:brand Brand Extract from a live URL, or invent one from scratch.
02 · /stardust:briefings Briefings One per page: intent, audience, key messages, tone.
03 · /stardust:wireframes Wireframes Grey, annotated structure. No colors, no type. Just shape.
04 · /stardust:design Design Branded HTML. Real fonts, real copy, iterated in the browser.
05 · /stardust:eds-design EDS design Design tokens, boilerplate CSS, block styling scaffolds.
06 · /stardust:eds-build EDS build Blocks and pages, wired into the aem-boilerplate project.
07 · /stardust:eds-refine EDS refine Lint, audit, PageSpeed. The boring part done on purpose.

Drive it

Each step is a skill. You invoke it when you want it.

Open Claude Code. Type /stardust:brand. Read the output. Edit the file. Type /stardust:briefings. Keep going.

Every skill writes to stardust/ in your repo. Inspect between steps. Re-run any of them. The pipeline resumes from whatever it finds on disk.

No magic. No one-shot. A pipeline that halts where you halt it.

The output

A real aem-boilerplate project.

No custom framework, no proprietary runtime, no vendor runtime tax. Stardust writes into an ordinary aem-boilerplate repo — the one you'd have started with anyway.

Every intermediate artifact stays on disk in stardust/. Review it, rewrite it, throw it away. The project still ships without Stardust installed.
