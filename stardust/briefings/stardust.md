# Intent
Convince a skeptical builder that Stardust is a real tool for going from a website need to a shipped EDS site — not another AI site-builder toy — and get them to the install command in under a minute.

# Audience
Two overlapping personas, both people who actually have to ship sites:
- **SLICC-adjacent devs** — already using or evaluating SLICC, comfortable with CLI tools and Claude Code, tired of framework churn.
- **EDS/AEM practitioners** — Adobe dev community, content authors, partners who know Edge Delivery and want faster project kickoff.
Primary for both: they need to build websites and want the design-to-site pipeline compressed into skills they can drive.

# Key Messages
1. Stardust is a set of skills that walks you from a need to a website — brand (if any), briefings, wireframes, design, blocks, pages. If the brand doesn't exist yet, Stardust helps invent it.
2. Each step is a skill you invoke on demand — guided, inspectable, re-runnable. Not a one-shot generator.
3. Built as a Claude Code plugin today, designed to run wherever skills run — SLICC and other agent hosts next.
4. Opinionated on EDS output: a real aem-boilerplate project, lintable and deployable, no lock-in.

# Calls to Action
- Primary: Install — copy the `claude plugin install` one-liner (currently `paolomoz/stardust`, swapping to `adobe/skills` at launch)
- Secondary: View on GitHub

# Tone
Inherit the SLICC deadpan-mechanical-mischievous voice. One mischief per screen max — the ice-cream vocabulary already carries the brand, Stardust shouldn't pile on a second metaphor. Copy is mechanical ("a skill per design step" > "empowering creative teams"). Strike-through rewrite motif allowed once, hero only. Stardust is a sibling product to SLICC on sliccy.com — same family, not a rebrand.
